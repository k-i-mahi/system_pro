\chapter{Introduction}

\section{Background}
There is still a tendency to divide academic workflows in most universities among several tools. Students tend to apply one system to.
courses, one more communication, and dedicated AI support platforms. This discontinuity produces redundant context.
alternating and poor continuity between instructional resources and learning support.

To overcome this, the project Cognitive Copilot is proposed as one platform that integrates LMS workflows and.
AI-assisted study. It is not aimed at substituting academic teaching, but to make available a structured digital space in which.
students and tutors are able to communicate on the same course information.
$ $
\section{Problem Statement}
The practical problems that drive the project are:
\begin{enumerate}[label=\arabic*.]
	item Course resources, routine, and communication are not integrated into a single workflow.
	|human|>item AI chat tools tend to be uncoupled with course material, leading to low-context responses.
	item CRUD behavior, validation and role-based interaction is inconsistently addressed in many systems, which offer partial features.
	item When architecture is not modular, deployment and maintenance are challenging.
\end{enumerate}

\section{Project Objectives}
The aims of this work are:
\begin{enumerate}[label=\arabic*.]
	item Installing a role-aware LMS with essential modules: authentication, courses, routine, community, notifications,
	analytics, settings, and profile.
	item Create an AI Tutor workspace with course/topic context, and study modes.
	item Support real world material workflows such as upload and retrieval-assisted question answering.
	item Provide well-defined request validation and authorization checks in the modules that have reliable CRUD operations.
	item Deployable full-stack Implementation: Provide a deployable full-stack implementation that can be used to develop and demonstrate locally.
\end{enumerate}
The Project has the following scope.
The scope implemented consists of:
\begin{itemize}
	item user authentication, profile and settings.
	item Management of courses, topics and materials.
	item Workflows of routine and attendance.
	item Community classes, community threads, community announcements, and community interactions.
	item Notification feed that supports real time updating.
	item Student and tutor analytics.
	item AI Tutor guided learning interaction.
\end{itemize}

This report has out-of-scope items such as mobile-native clients and institution-wide multi-tenant deployment policies.

\section{Project Significance}
Cognitive Copilot is important because it is an engineering integration. Rather than making LMS and AI independent of each other.
products, this project holds them in a single consistent system in which actions of AI are tied to course context and user role.
This gives a more useful academic resource and a superior software engineering artifact to use later.
