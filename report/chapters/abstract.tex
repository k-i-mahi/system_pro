\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

This report proposes a full-stack academic learning platform Cognitive Copilot, a combination of the basic Learning.
Work with an AI tutor through Management System (LMS). The project is designed based on realistic needs in the university:
course and topic administration, classroom community engagements, routine and attendance assistance, notifications, and
augmented study with AI. It is a web application based on the React frontend and FastAPI backend.

Its implementation is modular in nature. The front end offers student and tutor role based pages.
where the backend provides REST APIs to authenticate, courses, routine, community, analytics, settings, profile,
materials, and notifications. To enable the use of AI, the platform combines local model inference and retrieval-based.
AI Tutor workspace Quiz support and question answering. Asynchronous workers and OCR-aided ingestion are employed.
to maintain responsiveness in user interactions when dealing with long-running operations.

Project focuses on reliability engineering as opposed to experimental assertions. The report thus concentrates on implemented.
data flow, user workflow, modules, CRUD behaviors, validation and authorization design, and tested workflows. No unsupported
as final evidence, there are numbers of performance or accuracy. Rather, verification is reported using reproducible.
test and build processes on backend and frontend.

The concept of Cognitive Copilot illustrates how a project could be both academically interesting and implementation-oriented:
it provides an integrated learning experience, maintains AI functions within course context, and remains maintainable.
software design which could be extended to subsequent academic work.
