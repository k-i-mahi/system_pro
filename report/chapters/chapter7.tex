\chapter{Conclusions}

\section{Summary}
Cognitive Copilot is a usable academic platform that integrates core LMS workflows with an AI-powered tutor in a single web application. The project shows that modularity, schema backed APIs, role aware pages, and disciplined testing can support a feature set large enough to be realistic login, courses, routine, AI tutor, community, notifications, analytics, settings, and profile without collapsing into a brittle prototype.

\section{Limitations}
\begin{enumerate}[label=\arabic*.]
\item The project targets web workflows only; native mobile clients are out of scope.
\item Advanced AI behavior depends on local model availability and operator skill; we do not claim state of the art accuracy.
\item Enterprise hardening (multi tenant isolation at scale, full SOC style monitoring) is not implemented.
\item Formal user studies measuring learning gains were not part of this development cycle.
\end{enumerate}

\section{Recommendations and Future Work}
\begin{enumerate}[label=\arabic*.]
\item Expand automated tests for cross role and cross user permission boundaries until they match the breadth of the UI.
\item Continue polishing loading, empty, and error states and mutation feedback across all modules.
\item Add richer analytics exports and classroom reporting for tutors.
\item Improve production style deployment automation (secrets management, migrations, health checks).
\item Improve localization and accessibility for more inclusive education.
\end{enumerate}

How this work can be extended: a natural next step is institution specific identity integration, policy controlled AI usage, and hardened hosting building on the same modular boundaries we documented rather than rewriting from scratch.
