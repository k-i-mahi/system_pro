\section{Implementation Summary}
The implementation was carried out as a modular full-stack web application. The frontend was developed using React~19 and TypeScript with Vite~8. The backend was implemented with Express and TypeScript, Prisma ORM for schema modeling and migrations over PostgreSQL~15 with pgvector, Redis and BullMQ for asynchronous workers, and Ollama for local LLM responses. The AI sidecar uses Python~3.11, FastAPI, TrOCR, Nougat, and Unstructured \cite{react,express,prisma,postgres,redis,bullmq,ollama,tesseract,docker}.

\section{Module-Wise Implementation}

\subsection{Authentication and Profile}
JWT-based access (15 min TTL) and refresh token logic was implemented. Bcrypt password hashing is used server-side. Role-aware authorization middleware (\texttt{auth.ts}) guards all protected routes. The profile module includes avatar upload via Cloudinary, bio updates, and theme/date-format preferences.

\subsection{Course and Topic Management}
Course listing, topic CRUD, topic ordering, and material upload/link attachment are managed through dedicated controllers and Zod validators. An enrollment model tracks per-student course membership. Topic-level progress and exam attempt records feed the analytics and mastery dashboard.

\subsection{RAG Ask-Course Mode}
The \texttt{POST /api/ai-tutor/ask-course} endpoint receives a question and optional course/topic filter. It runs BM25 (via \texttt{websearch\_to\_tsquery}) and cosine search in parallel, fuses with RRF ($K=60$), picks the top-8 chunks as context, and streams the generation with \texttt{event: token} SSE frames. A final \texttt{event: citations} frame delivers the fully resolved citation objects (materialId, title, page, snippet) so the frontend can render clickable \texttt{CitationChip} badges without a round-trip.

\subsection{Agentic Chat Tutor}
The \texttt{POST /api/ai-tutor/chat} endpoint delegates to the ReAct loop (Section~\ref{sec:agent}). The strategy is selected deterministically before the first iteration. Every tool call and result is appended to the message context and logged to \texttt{LlmCall}. The SSE stream emits \texttt{event: strategy}, \texttt{event: thought}, \texttt{event: tool}, \texttt{event: token}, and \texttt{event: done} frames so the frontend can render a collapsible ``Tutor Thinking'' trace alongside the final answer.

\subsection{Adaptive Quiz}
\texttt{POST /api/ai-tutor/generate-quiz} uses Ollama's \texttt{format} parameter with a JSON schema for the five-question MCQ object, eliminating the regex parsing that caused 6--9 failures per 200 generations in the previous version. \texttt{POST /api/ai-tutor/submit-quiz} updates $(\alpha, \beta)$ on \texttt{TopicProgress} and returns the posterior mean and 95\% credible interval. Adaptive difficulty targets $d = \operatorname{clip}(\lceil 1 + 4\mu \rceil, 1, 5)$ on the next generation.

\subsection{Resource Discovery Mode}
\texttt{GET /api/ai-tutor/search-resources} queries Wikipedia's REST API, Stack Overflow search, Hacker News Algolia, and YouTube search simultaneously, classifies results by type (video / article / paper / blog), and returns a ranked list. The \texttt{ResourceSuggestPane} frontend component renders results with type icons and filter pills for All / Video / Article / Paper / Blog.

\subsection{OCR Routine Pipeline}
Routine scanning accepts timetable images, runs OCR extraction (fast or accurate path), parses probable course codes and time slots with a regex-based parser, and presents an editable draft to the user before bulk insertion. This prevents blind acceptance of noisy OCR output.

\subsection{Real-Time Notifications}
The notification service persists a row and emits \texttt{notification:new} and \texttt{notification:count} over Socket.IO (room \texttt{user:\$userId}) within the same synchronous call. The frontend \texttt{useNotificationSocket} hook subscribes and updates the TanStack Query cache instantly, so the bell badge reflects live state without polling.

\subsection{Analytics Dashboard}
\texttt{GET /api/analytics/overview} aggregates attendance, exam, and topic-progress data in a single response. \texttt{GET /api/analytics/evaluation} returns the instructor-facing LLM observability aggregate: p50/p95 latency per route, error counts, recent failures with drilldown, and per-topic Beta-posterior expertise curves.

\section{Grounded Retrieval-Augmented Q\&A}
\label{sec:rag-impl}
The RAG implementation (Section~\ref{sec:rag}) is the largest new deliverable. Four engineering decisions were load-bearing.

\begin{enumerate}[label=\arabic*.]
    \item \textbf{Shared persistence.} Embeddings live in a \texttt{vector(768)} column on the same Postgres instance that stores user data, avoiding a second database service for capstone-scale workloads.
    \item \textbf{Pagination-aware chunker.} The greedy packer targets 800 tokens with 150-token overlap and never crosses PDF page boundaries, which preserves page-number provenance for citations.
    \item \textbf{Hybrid fusion.} RRF with $K=60$ fuses sparse and dense rankings without requiring calibrated scores --- a practical win because cosine distance and \texttt{ts\_rank\_cd} are not directly comparable.
    \item \textbf{Streaming citations.} SSE emits tokens from the generator and a final \texttt{event: citations} frame. The frontend \texttt{renderWithCitations} helper splices them into \texttt{[$n$]} tokens without re-rendering the text buffer, making citations feel instant.
\end{enumerate}

\section{Agentic Tool Use and Socratic Strategies}
The chat controller delegates to the ReAct loop. Five strategies are implemented as prompt templates; a deterministic regex-based selector picks the strategy before generation starts, with a Socratic downgrade applied when the Beta-posterior mean falls below $0.4$. Every strategy transition, tool call, and tool result is logged to \texttt{LlmCall} with a shared \texttt{parentCallId}, which is how the evaluation dashboard reconstructs an agent trace.

Structured quiz output is produced through Ollama's JSON-schema-constrained \texttt{format} parameter rather than post-hoc regex parsing. Across 200 consecutive generations during stability testing, zero parse failures were observed, down from an average of 6--9 failures per 200 with the previous regex approach.

\section{Advanced OCR}
The backend's \texttt{extractTextFromFile} function accepts a \texttt{quality} parameter. The fast path preserves the original Tesseract and \texttt{pdf-parse} behaviour. The accurate path calls the Python FastAPI sidecar (TrOCR for handwriting, Nougat for academic PDFs with embedded mathematics, Unstructured for layout-heavy documents) via \texttt{undici} with FormData. All accurate-path calls flow through a dedicated BullMQ worker so slow inferences never block HTTP handlers.

Availability-first default: if the sidecar returns non-200 or the underlying model fails to load, the backend falls back to the fast path and surfaces an ``AI-transcribed --- please verify'' banner in the UI.

\section{Bayesian Mastery Tracking}
The conjugate Beta update described in Section~\ref{sec:mastery} replaces the previous $+0.1/-0.05$ heuristic. The hyperparameters $\alpha$ and $\beta$ are persisted on \texttt{TopicProgress}. After every quiz submission, the backend computes the posterior mean and 95\% credible interval and returns both to the client so the student can see their uncertainty explicitly. Adaptive difficulty uses the posterior mean to target the next quiz difficulty level.

\section{LLM Observability}
All Ollama callers flow through a \texttt{withLogging(route, fn)} higher-order function. Each call inserts a row into the \texttt{LlmCall} table capturing: route, model, latency, prompt and completion token counts, status (\texttt{OK}/\texttt{ERROR}/\texttt{TIMEOUT}), truncated error message, and \texttt{parentCallId} for agent trace reconstruction. Prometheus metrics are published at \texttt{/metrics} behind an \texttt{X-Metrics-Key} header: per-route latency histograms, token counters, and error rate gauges.

\section{Preliminary User Study}
A within-subjects pre/post study was conducted with volunteer KUET CSE undergraduates over a two-week window following the protocol in \texttt{study/protocol.md}. Participants completed a ten-item pre-test on two topics, used the tutor freely for one week, then took a parallel ten-item post-test plus a System Usability Scale (SUS) \cite{brooke1996sus,bangor2008sus} survey and a one-item net promoter question.

The paired t-test on quiz accuracy and the SUS summary are produced by \texttt{scripts/analyze\_study.py}. Effect size is reported as Cohen's $d$ on the differences. The fallback posture if recruitment yields fewer than six participants is documented in the protocol: preliminary self-case-study with the quantitative RAG and OCR metrics carrying the evaluative load. Full numerical tables and the pre/post boxplot are included in the award-submission packet when the study window closes.

\section{Discussion}
The upgraded implementation turns the previous generic LMS-plus-chat into a grounded, measurable, agent-driven tutor. Four outcomes stand out.

\textbf{Architecture resilience.} The route-controller-service separation inherited from the earlier iteration proved resilient: every new feature (RAG, sidecar OCR, evaluation harness, resource discovery) slots in behind an existing controller boundary without cross-cutting leaks.

\textbf{Sidecar isolation.} Moving all heavy Python ML into a FastAPI sidecar kept the backend Docker image small ($<$200\,MB) and the cold-start time predictable. The fallback path means a sidecar crash does not break OCR entirely.

\textbf{Zero-friction instrumentation.} The \texttt{withLogging} wrapper instrumented every LLM call without touching any caller's surface API. The evaluation dashboard, Prometheus endpoint, and \texttt{LlmCall} table all come for free.

\textbf{Interpretable mastery.} Adopting conjugate Bayesian updates turned what was previously an opaque heuristic into a statement a student can read: ``your mastery is 62\%, 95\% CI 47--76\%.'' The strategy badge gives a second interpretability layer: the student can see that the tutor chose Socratic probing because their posterior mean was below 0.4.

The main residual risks are the structured-output reliability of Qwen2.5~7B on edge cases (mitigated with a 200-generation regression test), CPU-only Nougat latency on live demos (mitigated by pre-computing a demo sample cache), and the study recruitment risk (mitigated by an honest n=6 reporting fallback).
